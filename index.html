I'm here to convince you of twothree things:

#1: You need standards

#2: Standards need you

#3: Standards can be fun!

The benefits to users of standards are fairly clear

Breaking down siloes, increasing efficiencies, driving innovation, facilitating FAIR principles, avoiding vendor lock-in...

Research into local government data sharing during covid highlighted need for standards and avoiding siloes etc

The benefits of users to standards are not so obvious

Standards should be demand-driven rather than supply-driven, and built as a consensus

Government involvement in the standards development process is valuable

Involvement in the development process ensures national priorities are addressed and that the public sector dimension is included

Standards can be a light-touch way of regulating rather than using legislation

In the UK Geospatial world standards are driven by ISO and OGC

Both have different approaches but both have a strong requirement for users

BSI/ISO have been more traditional but are developing a fast track approach

The basic process is:

Propose -> Assemble Panel -> Draft -> Comment -> Refine -> Approve -> Publish

People are needed at every stage- is there a need for the standard, do we have people with domain knowledge? Does the standard already exist elsewhere? Is it clear? Are all the use cases considered?

Proposing- or not reinventing the wheel

Catching standards that replicate something that already exists is vital

Reference to ISO Indoor Feature Model versus OGC Indoor Mapping Data Format

Assembling domain experts

Some standards are passed with no UK input because we don't have domain experts that can comment for us

Indulge your inner pedant

You can do a lot by commenting on one of the draft standards

Generally via a templated email or word document.

Refine

This could be making documentation clearer, or actually adding to the standard

UK Gemini - lots of work around improving guidance, but also adding in things like Metadata Standard Name and Version, which are needed for the profile to be properly used inside GeoNetwork

Rinse and Repeat

The draft -> comment -> refine process can go on for several iterations

Often 3 years or so

Input from actual users or implementers is vital!

Some things that look good on paper don't work IRL

Example UK Gemini true/false/not tested for conformance

Convinced?

To summarise:

  • There are a lot of steps between metadata standard and plugin
  • These steps all introduce complexities and require implementation decisions
  • Profile plugins can be used to enhance the user experience, and make it easier for them to produce high-quality metadata that meets the standard
